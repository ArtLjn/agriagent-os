# 08 — 业务模块化

> 状态：已按 2026-07-24 代码现状校准 | 维护：BlockShip | 关联：[01_Agent平台架构](./01_Agent平台架构.md)、[../../../docs/architecture/boundaries.md](../../../docs/architecture/boundaries.md)

---

## 1. 当前结论

旧文档描述的 `app/api` + `app/services` + `app/models` 散落结构已经下线，目标也不再是 `modules/<domain>/`。当前后端按 **业务领域 domains + 平台能力 platforms + Agent 平台 + shared/infra 基础设施** 分层。

新增代码不得重新创建 `app.api`、`app.models`、`app.schemas`、`app.services`、`app.modules`、`app.core`、`app.simulation` 旧入口。需要新增业务能力时，优先放入已有 `domains/*` 或 `platforms/*` 边界。

## 2. 当前目录结构

```text
backend/app/
├── bootstrap/       # 应用工厂、路由注册、中间件、异常、lifespan
├── application/     # chat/session/advice/report/smart_fill 等 use case
├── agent/           # router/runtime/executor/reflector/guardrails/planning
├── context/         # selector、budget、compressor、cache、renderer
├── memory/          # short_term、long_term、consolidation、service
├── prompt/          # registry、composer、renderer、replay、cache
├── skills/          # Skill 包、metadata、registry YAML、执行适配
├── domains/
│   ├── users/       # 认证、用户资料、用户设置、配额、后台用户管理
│   ├── farm/        # 农场模型、当前农场依赖、农场上下文与报表数据
│   ├── planting/    # 作物模板、茬口、农事日志、地块、工单、工人、工资
│   ├── finance/     # 成本、分类、债务
│   ├── weather/     # 天气服务、地点解析、QWeather/Open-Meteo provider
│   └── conversation/# Agent HTTP、会话、日报建议、报告、反馈
├── platforms/
│   ├── admin/       # 后台配置、Dashboard、Trace、版本、Token 统计
│   ├── data_flywheel/ # 样本、标注、预标注、问题链、repair pack
│   ├── evaluation/  # 回放、指标、Discovery、报告、诊断
│   ├── simulation/  # Agent 仿真 API、runner、reporter、consistency checker
│   └── shared/      # judge service 等跨平台共享协议
├── shared/          # config、database、logging、security、time、schema、LLM manager
├── infra/           # Mongo、repository runtime、trace、limiter、pending、cache、RAG client
├── ops/             # seed、schema hardening audit 等运维辅助
└── observability.py # 平台观测事件入口
```

## 3. 领域边界

| 领域 | 真实路径 | 主要职责 | 典型 HTTP |
| --- | --- | --- | --- |
| Users | `domains/users/` | 注册、登录、当前用户、用户设置、配额、后台用户管理 | `/auth/*`, `/settings`, `/admin/users/*` |
| Farm | `domains/farm/` | 农场模型、当前农场解析、农场上下文、报告数据 | 通过 Depends / service 被其他领域调用 |
| Planting | `domains/planting/` | 作物模板、系统模板库、茬口、地块、农事日志、工单、工人、工资 | `/crops/*`, `/cycles/*`, `/logs`, `/planting/*`, `/smart-fill/*` |
| Finance | `domains/finance/` | 成本记录、成本分类、赊账与结算 | `/costs`, `/cost-categories`, `/debts` |
| Weather | `domains/weather/` | 天气预报、预警、地点搜索、城市/地域解析 | `/weather/forecast`, `/locations/*` |
| Conversation | `domains/conversation/` | Agent Chat/SSE、会话列表、日报建议、报告、反馈 | `/agent/*` |

## 4. 平台边界

| 平台 | 真实路径 | 主要职责 |
| --- | --- | --- |
| Admin | `platforms/admin/` | Skill/Prompt 配置、Trace 查询、Dashboard、Token 统计、版本接口 |
| DataFlywheel | `platforms/data_flywheel/` | 样本列表、人工标注、AI 预标注、session 同步、问题链、repair pack |
| Evaluation | `platforms/evaluation/` | 回放用例、指标、Discovery 风险发现、诊断报告、Prompt compare |
| Simulation | `platforms/simulation/` | 仿真 run API、Agent client、状态快照、语义提取、一致性检查、报告 |
| Shared | `platforms/shared/` | DataFlywheel/Evaluation 共用 judge client 与共享协议 |

平台能力可以依赖领域依赖和共享基础设施，但不应复制领域业务规则。例如 DataFlywheel 可以读取会话/trace 证据，但成本、茬口、工单等业务口径仍由对应 domain service 保证。

## 5. 依赖方向

```text
bootstrap/routes
  -> domains/*/routes、platforms/*/routes、application/*
  -> domains/*/service、agent/*、context/*、memory/*、prompt/*、skills/*
  -> platforms/shared、shared、infra
```

约束：

- `domains/*/*routes.py` 只做 HTTP 编排、Depends、schema 转换和 service/use case 调用。
- `application/` 可以编排 Agent Runtime、Context、Prompt、Memory 和领域服务，但不直接承载 HTTP request 细节。
- `agent/runtime/` 不直接查询数据库，不直接 import `context/selectors` 或 `memory/long_term` 存储实现。
- `skills/` 是 Agent 调业务能力的翻译层；需要 DB 时通过注入的 session 和领域 service/脚本适配完成，不复制 HTTP 层逻辑。
- `shared/` 不依赖业务领域；`infra/` 只放外部系统、存储、trace、限流、repository runtime 等适配。

## 6. Skill 与业务领域的关系

```text
LLM 选择工具
  -> agent/router 选择候选 Skill
  -> agent/executor 处理 tool_call / pending / 参数
  -> skills/<skill-name>/scripts/main.py
  -> domains/<domain>/service.py
  -> ORM model / repository runtime
```

Skill 层关注自然语言到业务参数的转换、写操作确认、错误转中文反馈和 tool result 结构。领域 service 负责业务规则，例如赊账结算不重复计入利润、系统作物模板导入使用副本模式、工人工资同步人工成本账单。

## 7. 当前状态

- ✅ `domains/users|farm|planting|finance|weather|conversation` 已是当前生产代码边界。
- ✅ `platforms/admin|data_flywheel|evaluation|simulation|shared` 已是当前平台能力边界。
- ✅ `skills/` 已从旧 `agent/skills/` 迁到平台级 `app/skills/`。
- ✅ `app.core/api/models/schemas/services/modules/simulation` 旧入口不再作为活动 import 路径。
- 🚧 后续工作主要是继续压缩大文件、减少跨域直接 service 调用、补齐边界检查脚本，而不是迁移到 `modules/`。
