# 视觉 QA 清单

> 使用方式：UI 实现完成后逐项检查。发现问题时记录 severity、截图路径、复现步骤和建议修复点。

## 1. 基础可用性

- [ ] 页面能正常打开，无崩溃、白屏、死循环加载。
- [ ] 用户能完成本次任务定义的核心操作。
- [ ] 返回、取消、保存、提交等关键操作行为符合预期。
- [ ] 异步操作有明确反馈，不让用户误以为无响应。
- [ ] 危险操作有确认或可撤销设计。

## 2. 信息层级

- [ ] 首屏能看出页面主任务。
- [ ] 主标题、核心数据、主按钮之间层级清楚。
- [ ] 次要信息没有抢主操作。
- [ ] 页面没有无意义装饰元素。
- [ ] 同类信息使用一致的排版和组件。

## 3. 布局和适配

- [ ] 小屏 Android 下内容不横向溢出。
- [ ] 长文本不会压坏卡片、按钮、表单或列表。
- [ ] 键盘弹出后输入框和提交按钮仍可访问。
- [ ] 滚动区域清楚，没有内外滚动冲突。
- [ ] 底部导航、悬浮按钮和系统安全区不遮挡内容。
- [ ] 横屏或异常尺寸下至少保持可用。

## 4. 状态覆盖

- [ ] 默认状态完整。
- [ ] loading 状态清楚且不闪烁。
- [ ] empty 状态说明下一步动作。
- [ ] error 状态包含可恢复操作。
- [ ] offline/弱网状态不导致误提交。
- [ ] 表单校验错误靠近对应字段。
- [ ] 长列表、分页或无更多数据状态明确。

## 5. 视觉一致性

- [ ] 颜色来自 `FarmManagerMobile/src/theme/colors.ts`。
- [ ] 渐变来自 `FarmManagerMobile/src/theme/colors.ts` 中的 `gradients`。
- [ ] 圆角来自 `FarmManagerMobile/src/theme/designTokens.ts` 中的 `radii`。
- [ ] 字号和字重遵循 `typography`。
- [ ] 阴影遵循 `shadowV2`。
- [ ] 动效时长和幅度遵循 `animationTiming`。
- [ ] 没有新增一次性色值、魔法间距或临时样式。
- [ ] 实现结果符合 `docs/ui/design-to-code-handoff.md` 中的结构拆解。
- [ ] 高风险视觉效果已按交接清单采用首选实现或降级方案。

## 6. 移动端触控

- [ ] 可点击区域不小于 44px。
- [ ] 列表项、按钮、图标按钮有 pressed/disabled 状态。
- [ ] 表单输入顺序自然。
- [ ] 误触高风险操作有足够间距。
- [ ] 横向滑动、纵向滚动不会互相干扰。

## 7. 文案和可理解性

- [ ] 文案使用用户能理解的农业/记账语言。
- [ ] 错误文案说明原因和下一步。
- [ ] 按钮文案是动作，不是抽象名词。
- [ ] 没有 mock、TODO、placeholder 文案。
- [ ] 金额、日期、单位、天气数据格式一致。

## 8. 审美污染检查

- [ ] 没有为了"高级感"新增大面积渐变背景。
- [ ] 没有卡片套卡片导致层级混乱。
- [ ] 没有过多 chip/tag 装饰。
- [ ] 没有无意义 emoji 堆叠。
- [ ] 没有一屏过多颜色同时抢注意力。
- [ ] 没有过强阴影、过大圆角或悬浮感滥用。
- [ ] 没有与 `docs/ui/app/ui.md` 冲突的视觉语言。

## 9. 截图要求

每次 UI 任务至少保留：

- [ ] 默认状态截图。
- [ ] loading 或提交中截图。
- [ ] empty 或 error 截图。
- [ ] 小屏截图。
- [ ] 涉及表单时，键盘弹出截图。

推荐路径：

```text
FarmManagerMobile/docs/screenshots/YYYY-MM-DD/<task-name>-<state>.png
```

## 10. 结论格式

验收完成后，按下面格式回复：

```text
视觉 QA 结论：通过 / 部分通过 / 未通过

已验证：
- 页面：
- 设备/尺寸：
- 截图：
- 命令：

发现的问题：
- [P1] 问题描述，截图路径，建议修复方式

剩余风险：
- 未覆盖的状态或设备
```
