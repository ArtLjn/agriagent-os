## Context

当前账单系统以 `cost_records` 为主表，普通支出、收入、赊账、还款和人工成本都落在同一张表。移动端账单总览按 `record_type=cost/income` 汇总，导致赊账支出被显示成已支出现金；还款又被建成收入类记录，进一步干扰收入和利润理解。用户明确希望保持轻量记事本定位，不引入支付对接或完整会计系统。

## Goals / Non-Goals

**Goals:**
- 在保留 `cost_records` 主模型的前提下，补齐轻量结算状态。
- 让账单总览同时展示发生额、已结金额和未结金额，避免把赊账误认为已付款。
- 让赊账还款/应收收款更新原账单状态，而不是作为普通收入污染汇总。
- 保持人工工资、作业单人工成本继续进入利润成本发生额，同时对已付/未付口径可解释。

**Non-Goals:**
- 不引入复式记账、会计科目、凭证、银行账户或支付渠道。
- 不拆分账本到多张复杂财务表。
- 不处理税务、发票、合同、人事薪资等专业财务流程。
- 不要求历史账单全量精确还原真实现金流，只做可解释迁移。

## Decisions

1. 保留 `cost_records`，增加最小结算字段。
   - 选择：新增 `settled_amount`、`settlement_status`，未结金额由 `amount - settled_amount` 推导或响应层返回。
   - 理由：当前账单是记事本，单表增强成本最低，也能解决赊账误扣现金的问题。
   - 替代方案：新增 settlement 表记录每次还款。放弃原因是当前阶段过度设计，用户只需要可靠状态，不需要付款明细审计。

2. `amount` 表示发生金额，不再表示已付/已收现金。
   - 选择：利润和茬口成本继续按发生额统计；现金总览按 `settled_amount` 统计。
   - 理由：赊账买农资确实是成本发生，但不是现金已经支出，两个口径必须分开。
   - 替代方案：赊账不进入成本直到还款。放弃原因是会让利润低估已发生成本。

3. 还款/收款更新原未结账单。
   - 选择：`/debts/settle` 按对方或记录定位未结账单，增加 `settled_amount`，同步 `settlement_status` 和 `settled_at`。
   - 理由：用户关心“这笔账还欠多少”，原账单状态比新增一条收入更直观。
   - 替代方案：继续创建还款记录但从收入汇总排除。放弃原因是列表仍会出现误导性收入项，且需要更多筛选规则。

4. 账单总览改为三组轻量指标。
   - 选择：展示本月发生支出/收入、本月已付/已收、当前未付/未收。
   - 理由：不需要复杂报表，就能把用户最困惑的状态讲清楚。
   - 替代方案：只在赊账列表里展示未结。放弃原因是总览仍会制造“钱扣出去了”的错觉。

5. 人工工资复用同一结算口径。
   - 选择：人工成本账单的 `amount` 仍为应付总额，`settled_amount` 对齐已付金额，`settlement_status` 对齐未付状态。
   - 理由：人工应付是成本发生，已付/未付又是现金状态，和赊账问题本质相同。

## Risks / Trade-offs

- [Risk] 历史还款记录已作为收入存在，迁移后可能与原赊账重复。→ 迁移阶段优先标记/排除旧还款类记录，必要时只对新数据启用新口径。
- [Risk] `settled_amount` 与人工 `paid_amount` 形成冗余。→ 人工工资更新必须通过现有 labor service 同步成本账单，测试覆盖一致性。
- [Risk] 部分还款按对方自动分摊可能不符合用户预期。→ 初期沿用当前最早未结账单优先规则，并在响应中返回受影响记录。
- [Risk] 前端文案不清会继续误导。→ 总览和列表使用“发生”“已付/已收”“未付/未收”固定文案，不再只展示单一结余。

## Migration Plan

1. 数据库新增 `settled_amount` 和 `settlement_status`，默认历史普通账单为已结清。
2. 历史 `record_subtype='赊账'` 且未结清记录迁移为 `settled_amount=0`、`settlement_status='unsettled'`。
3. 历史已 `settled_at` 的赊账记录迁移为 `settled_amount=amount`、`settlement_status='settled'`。
4. 旧 `category='还款'` 且带 `parent_record_id` 的收入记录不再参与普通收入汇总；可先软排除新聚合，后续再做数据清理。
5. 后端 API 先兼容旧字段，新增结算字段；移动端切换展示口径后再考虑隐藏旧还款记录。
