# Proposal: AI 长时记忆架构

## 背景

当前 Agent 只能被动调用离散的 Skills 回答问题，无法像人类顾问那样"记得"用户的完整种植历史、收支习惯和农场状态。用户需要一个能主动理解其全量数据、给出聚合洞察的 AI 助手。

## 目标

让 Agent 具备用户全量数据的长期上下文，支持：
- **精确聚合查询**："这个月人工花了多少" → SQL 精确计算
- **语义关联查询**："西瓜最近怎么样" → 跨数据源整合
- **趋势洞察**："成本比去年多了吗" → 时间序列分析
- **多用户隔离**：为后续多农场主接入预留空间

## 演进路线

### Phase 1: 增强 Skills（当前）
- 增强 `cost-summary` Skill：支持按分类/按月/按类型/全局查询
- 新增 `cost-analytics` Skill：跨周期聚合分析
- 所有 Skill 支持 `farm_id` 过滤（多用户预留）

### Phase 2: 记忆摘要
- 定时任务生成用户数据画像摘要
- 对话开始时将摘要注入 System Prompt
- 增量更新机制

### Phase 3: 向量语义检索
- 农事记录、成本记录向量化索引
- 支持模糊语义搜索
- 向量库按 `farm_id` metadata 隔离

## 范围

- 后端：Skills 系统增强 + 新 Skill 开发
- 前端：Agent 聊天界面展示结构化数据
- 不涉及：LLM 模型更换、数据迁移

## 非目标

- 不更换底层 LLM（继续使用 DashScope）
- 不修改数据库 Schema（已有 `farm_id`）
- 不引入外部向量库（Phase 3 再考虑 pgvector/Chroma）
