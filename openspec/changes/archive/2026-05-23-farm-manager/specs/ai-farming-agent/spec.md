## ADDED Requirements

### Requirement: Agent 收集种植上下文信息
系统 SHALL 在生成建议前，自动收集当前茬口的周期状态、近期天气、近期农事记录等上下文信息。

#### Scenario: 上下文收集
- **WHEN** Agent 被触发生成建议
- **THEN** 系统收集：当前阶段、距下阶段天数、最近3天天气、最近7天农事记录、近期成本支出

### Requirement: Agent 生成个性化农事建议
系统 SHALL 基于收集的上下文，通过 AI 大模型生成个性化农事建议，包括待办事项、注意事项、优化建议。

#### Scenario: 每日建议生成
- **WHEN** 每天早上8点 Agent 定时运行
- **THEN** 系统生成并推送当日农事建议，如"今日晴，适合授粉；注意第3棚有3株长势偏弱，建议追肥"

#### Scenario: 用户主动询问
- **WHEN** 用户在 App 中输入"西瓜叶子有点黄怎么办"
- **THEN** Agent 结合近期农事和天气记录，给出可能原因和解决方案

### Requirement: Agent 生成周/月总结报告
系统 SHALL 定期（每周/每月）生成种植总结报告，回顾本周/月农事完成情况、成本支出、下阶段计划。

#### Scenario: 周报生成
- **WHEN** 每周日晚上 Agent 运行
- **THEN** 系统生成周报："本周完成授粉3棚，支出肥料费500元；下周进入膨果期，建议准备钾肥"

### Requirement: Agent 建议标注免责声明
系统 SHALL 在所有 AI 生成的建议中明确标注"建议仅供参考，请结合实际情况判断"。

#### Scenario: 建议展示
- **WHEN** 用户查看 Agent 生成的建议
- **THEN** 每条建议下方显示免责声明
