## ADDED Requirements

### Requirement: Write operations use single LLM call
Write operation skills SHALL require only one LLM invocation. The confirmation message SHALL be generated by the system template, not by a second LLM call.

#### Scenario: Cost record with single LLM call
- **WHEN** the user says "买了200块化肥"
- **THEN** the system SHALL invoke the LLM exactly once
- **AND** the LLM SHALL return a tool_call for `create_cost_record`
- **AND** the tool_node SHALL intercept it and return a confirmation message
- **AND** the LLM SHALL output the confirmation message directly
- **AND** the user SHALL see the confirmation within 2 seconds (previously 4-6 seconds)

#### Scenario: No second LLM round for pending actions
- **WHEN** a write skill is intercepted as pending action
- **THEN** the system SHALL NOT make a second LLM call to generate the confirmation text
- **AND** the confirmation text SHALL come from `build_confirm_message()` template

### Requirement: System prompt cached by farm and date
The rendered system prompt SHALL be cached with a key derived from `farm_id`, `current_date`, and `template_version`. Cache hits SHALL skip database queries and template rendering.

#### Scenario: Cache hit for repeated requests
- **WHEN** a user makes a second request on the same day for the same farm
- **THEN** the system SHALL use the cached system prompt
- **AND** the request SHALL not query farm information from the database again
- **AND** the response latency SHALL be reduced by at least 100ms

#### Scenario: Cache miss on new day
- **WHEN** a user makes a request on a new day
- **THEN** the cache key SHALL miss (date changed)
- **AND** the system SHALL render a fresh system prompt
- **AND** the new prompt SHALL be cached for subsequent requests

### Requirement: Model routing by task complexity
The system SHALL route LLM requests to different model tiers based on input classification: lightweight models for greetings, standard models for regular queries, and premium models for complex analysis.

#### Scenario: Greeting uses lightweight model
- **WHEN** the user says "你好" or "在吗"
- **THEN** the system SHALL use the lightweight model (e.g., Qwen-Turbo)
- **AND** the response latency SHALL be under 1 second

#### Scenario: Complex query uses premium model
- **WHEN** the user says "帮我分析今年种植方案"
- **THEN** the system SHALL use the standard or premium model
- **AND** the system SHALL NOT downgrade to lightweight model
